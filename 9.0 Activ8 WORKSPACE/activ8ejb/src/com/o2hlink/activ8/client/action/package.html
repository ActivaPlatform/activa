Action package

The command pattern is used in order to make tolerable the asynchronous request made by the client.

It is the leverage point for

  • Caching: By keeping a heap of the last actions performed.
  • Batching: By sending all the actions in a list of actions.
  • Centralize failure handling: By implementing a class which dispatch the actions.

It also lays the background for

  • Load client code on demand
  • Undo/Redo: By keeping an stack of actions
  • Google Gears/HTML5 DB: Storing actions will help the client to work disconnected from the internet

Actions are request sent to the server. Actions must implement the action interface

	/**
	 * ..
	 **/
	public interface Action<R extends Response> extends Serializable {
	}

Optionally they can implement the Cacheable interface. The client is in charge of doing the cache in order to avoid calls to the server.

	/**
	 * ..
	 **/
	public interface Cacheable {
		/**
		 * @return The time in milliseconds to which the last result will be cached
		 **/
		public long getMaxAge();
	}
